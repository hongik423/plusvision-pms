# 플러스비젼 PMS - Use Case 문서

> **버전:** 1.0 | **작성일:** 2026-02-24

---

## 1. 액터 정의

| 액터 | 설명 |
|------|------|
| **관리자 (Admin)** | 시스템 전체 관리, 최봉 대표 및 지정 관리자 |
| **매니저 (Manager)** | 프로젝트 총괄, 담당자 배정 |
| **사용자 (User)** | 엔지니어, 영업 담당 등 실무자 |
| **조회자 (Viewer)** | 읽기 전용 (외부 또는 제한된 접근) |
| **시스템 (System)** | 자동 알림, 로그 기록, 폴더 생성 등 |

---

## 2. Use Case 목록

### UC-01: 프로젝트 생성

| 항목 | 내용 |
|------|------|
| **액터** | Manager, Admin |
| **선행 조건** | 로그인 상태, 생성 권한 보유 |
| **트리거** | [+ 새 프로젝트] 버튼 클릭 |

**기본 흐름:**
1. 사용자가 프로젝트 생성 페이지에 접근한다
2. 시스템이 유사 프로젝트 추천 목록을 표시한다 (사업장/공정/품목 기반)
3. 사용자가 신규 생성 또는 기존 프로젝트 복사를 선택한다
4. 사용자가 필수 정보를 입력한다 (프로젝트명, 고객사, 사업장, 공정, 품목)
5. 사용자가 단계별 담당자를 지정한다 (선택사항, 이후 배정 가능)
6. 사용자가 [생성] 버튼을 클릭한다
7. 시스템이 프로젝트를 생성하고 표준 10단계를 자동 생성한다
8. 시스템이 파일 저장소에 프로젝트 폴더를 자동 생성한다
9. 프로젝트 상세 페이지로 이동한다

**대안 흐름:**
- 3a. 유사 프로젝트 복사 선택 시:
  - 기존 프로젝트의 기본정보, 부품 리스트, 견적 항목이 복사된다
  - 사용자가 복사된 내용을 수정한다
- 6a. 필수 입력 누락 시 검증 오류 표시

**후행 조건:** 프로젝트가 "대기" 상태로 생성, 1단계 담당자에게 알림

### UC-02: 프로젝트 단계 진행

| 항목 | 내용 |
|------|------|
| **액터** | User, Manager |
| **선행 조건** | 해당 단계의 담당자이거나 Manager 이상 |
| **트리거** | 현재 단계 카드 클릭 |

**기본 흐름:**
1. 사용자가 프로젝트 상세 페이지에서 현재 진행 단계를 클릭한다
2. 시스템이 해당 단계의 상세 정보와 필요 액션을 표시한다
3. 사용자가 필요한 문서를 업로드하거나 정보를 입력한다
4. 사용자가 [단계 완료] 버튼을 클릭한다
5. 시스템이 해당 단계를 "완료"로 변경하고 완료일을 기록한다
6. 시스템이 다음 단계를 "진행중"으로 활성화한다
7. 시스템이 다음 단계 담당자에게 알림을 발송한다

**대안 흐름:**
- 4a. 이전 단계가 미완료인 경우 완료 처리 불가 (경고 표시)
- 4b. 4단계(진행 여부 결정)에서 "보류/취소" 선택 가능
  - 보류: 프로젝트 상태를 "보류"로 변경
  - 취소: 프로젝트 상태를 "취소"로 변경, 사유 입력 필수

### UC-03: 문서 등록 및 조회

| 항목 | 내용 |
|------|------|
| **액터** | User, Manager, Admin |
| **선행 조건** | 해당 프로젝트 접근 권한 |
| **트리거** | 문서 탭 클릭 또는 단계 상세에서 문서 업로드 |

**기본 흐름:**
1. 사용자가 프로젝트 문서 탭에 접근한다
2. 시스템이 단계별로 그룹핑된 문서 목록을 표시한다
3. 사용자가 [파일 업로드] 버튼을 클릭한다
4. 사용자가 단계, 문서 유형을 선택하고 파일을 첨부한다
5. 시스템이 파일을 저장하고 목록에 추가한다
6. 감사 로그에 업로드 기록이 남는다

**대안 흐름:**
- 3a. 기존 파일 클릭 시 미리보기/다운로드 제공
- 4a. 지원하지 않는 파일 형식이면 오류 표시
- 4b. 파일 크기 초과 시 오류 표시

### UC-04: 견적서 작성

| 항목 | 내용 |
|------|------|
| **액터** | User, Manager |
| **선행 조건** | 프로젝트가 6단계(견적 작성) 이상 진행 |
| **트리거** | 견적 탭에서 [견적서 작성] 클릭 |

**기본 흐름:**
1. 사용자가 견적 탭에서 [견적서 작성]을 클릭한다
2. 시스템이 프로젝트 기본 정보를 자동 입력한 견적서 폼을 표시한다
3. 사용자가 부품을 마스터 데이터 드롭다운에서 선택한다
4. 사용자가 수량, 단가를 입력한다 (합계 자동 계산)
5. 사용자가 비고란에 추가 사항을 입력한다
6. 사용자가 [저장] 또는 [PDF 내보내기]를 클릭한다
7. 시스템이 견적서를 저장하고 해당 단계 문서로 자동 연결한다

**대안 흐름:**
- 3a. 마스터 데이터에 없는 부품은 직접 입력 가능 (자유 입력 필드)
- 6a. PDF 내보내기 시 회사 로고·양식이 포함된 공식 견적서 생성

### UC-05: 유사 프로젝트 검색 및 참조

| 항목 | 내용 |
|------|------|
| **액터** | User, Manager, Admin |
| **선행 조건** | 로그인 상태 |
| **트리거** | 프로젝트 생성 시 또는 검색 페이지 접근 |

**기본 흐름:**
1. 사용자가 사업장, 공정 유형, 품목 유형 등으로 검색한다
2. 시스템이 일치하는 프로젝트 목록을 반환한다
3. 사용자가 특정 프로젝트를 클릭하여 상세 내용을 확인한다
4. 사용자가 해당 프로젝트의 견적서, 부품 리스트, 매뉴얼 등을 참조한다
5. 필요 시 [이 프로젝트 복사하여 신규 생성] 클릭

---

### UC-06: 담당자 배정 및 업무 확인

| 항목 | 내용 |
|------|------|
| **액터** | Manager, Admin |
| **선행 조건** | 프로젝트 존재, 배정 권한 |
| **트리거** | 단계별 담당자 지정 액션 |

**기본 흐름:**
1. 관리자/매니저가 프로젝트 상세에서 특정 단계의 담당자를 지정한다
2. 시스템이 해당 사용자에게 알림을 발송한다
3. 담당자가 대시보드의 "My 할일"에서 배정된 업무를 확인한다
4. 담당자가 해당 프로젝트·단계를 클릭하여 상세 내용을 확인한다

---

### UC-07: 마스터 코드 관리

| 항목 | 내용 |
|------|------|
| **액터** | Admin |
| **선행 조건** | 관리자 권한 |
| **트리거** | 관리 > 마스터 코드 메뉴 접근 |

**기본 흐름:**
1. 관리자가 마스터 코드 관리 페이지에 접근한다
2. 관리 카테고리를 선택한다 (사업장/공정/품목/부품/고객사)
3. 기존 항목 목록이 표시된다
4. 관리자가 항목을 추가/수정/비활성화한다
5. 변경 내용이 즉시 반영되어 드롭다운 목록에 적용된다

---

### UC-08: 데이터 변경 이력 조회

| 항목 | 내용 |
|------|------|
| **액터** | Admin |
| **선행 조건** | 관리자 권한 |
| **트리거** | 프로젝트 이력 탭 또는 관리 > 로그 |

**기본 흐름:**
1. 관리자가 특정 프로젝트의 이력 탭을 클릭한다
2. 시스템이 해당 프로젝트의 모든 변경 기록을 시간순으로 표시한다
3. 각 기록에는 일시, 사용자, 변경 항목, 변경 전/후 값이 포함된다
4. 관리자가 특정 기록을 클릭하면 상세 변경 내용을 확인할 수 있다

---

### UC-09: 인수인계

| 항목 | 내용 |
|------|------|
| **액터** | Manager, Admin |
| **선행 조건** | 담당자 변경 필요 |
| **트리거** | 담당자 교체 액션 |

**기본 흐름:**
1. 관리자가 특정 프로젝트/단계의 담당자를 변경한다
2. 시스템이 이전 담당자의 모든 데이터·문서를 그대로 유지한다
3. 새 담당자에게 알림이 발송된다
4. 새 담당자가 프로젝트 상세에서 전체 진행 이력과 문서를 확인한다
5. 담당자 변경 이력이 감사 로그에 기록된다

---

### UC-10: 대시보드 조회

| 항목 | 내용 |
|------|------|
| **액터** | 모든 로그인 사용자 |
| **선행 조건** | 로그인 상태 |
| **트리거** | 로그인 후 기본 페이지 또는 대시보드 메뉴 클릭 |

**기본 흐름:**
1. 사용자가 대시보드에 접근한다
2. 시스템이 다음 정보를 표시한다:
   - My 할일: 나에게 배정된 진행중 업무 목록
   - 프로젝트 현황: 단계별 분포 차트 (파이/바 차트)
   - 최근 활동: 최근 등록/수정/완료된 프로젝트
   - 월별 실적: 완료 프로젝트 수, 매출 실적
3. 각 항목을 클릭하면 해당 상세 페이지로 이동한다
