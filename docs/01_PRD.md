# 플러스비젼 프로젝트 관리 시스템 (PlusPMS) - PRD

> **버전:** 1.0
> **작성일:** 2026-02-24
> **대상:** Cursor AI 개발 환경

---

## 1. 제품 개요

### 1.1 제품명
**PlusPMS** (Plus Vision Project Management System)

### 1.2 한줄 요약
반도체 공정 설비 제작·수리·설치 전문기업 플러스비젼의 프로젝트 라이프사이클을 10단계로 표준화하여 관리하는 웹 기반 프로젝트 관리 시스템

### 1.3 핵심 가치
1. **문서 중앙 관리** — 프로젝트 관련 문서를 체계적으로 축적·보관
2. **업무 명확화** — 각 담당자가 현재 자신의 할 일을 즉시 인지
3. **정보 재활용** — 유사 프로젝트 참조로 반복 작업 제거, 시간 단축

### 1.4 대상 사용자
| 역할 | 설명 | 주요 사용 기능 |
|------|------|---------------|
| 대표/관리자 | 전체 프로젝트 현황 파악, 권한 관리 | 대시보드, 권한 설정, 마스터 코드 관리 |
| 프로젝트 매니저 | 프로젝트 생성·배정·진행 관리 | 프로젝트 생성, 단계 관리, 담당자 지정 |
| 엔지니어 | 제작·설치 실무, 문서 작성 | 문서 등록, 단계 완료 처리, My 할일 |
| 영업/견적 담당 | 견적·제안서 작성 | 견적 작성, 제안서 생성, 유사 프로젝트 참조 |

## 2. 기술 스택

| 구분 | 기술 | 선정 사유 |
|------|------|-----------|
| **Frontend** | Next.js 14 (App Router) + TypeScript | SSR, 직관적 라우팅, 빠른 개발 |
| **UI 라이브러리** | Tailwind CSS + shadcn/ui | 일관된 디자인, 커스터마이징 용이 |
| **Backend** | Next.js API Routes + Prisma ORM | 풀스택 단일 프로젝트, 타입 안전성 |
| **Database** | PostgreSQL (Supabase) | 클라우드 자동 백업, Row Level Security |
| **인증** | NextAuth.js (Credentials + Google) | 구글 계정 연동, 역할 기반 접근 |
| **파일 저장** | Supabase Storage | S3 호환, 프로젝트별 폴더 자동 생성 |
| **검색** | PostgreSQL Full-Text Search | 별도 검색 엔진 불필요, 한국어 지원 |
| **배포** | Vercel | Next.js 최적화, 자동 배포 |
| **상태 관리** | Zustand | 가볍고 직관적 |

## 3. 핵심 기능 명세

### 3.1 프로젝트 관리 (Priority 1)

#### F-001: 프로젝트 CRUD
- 신규 프로젝트 생성 시 표준 10단계 자동 생성
- 필수 입력: 프로젝트명, 고객사, 사업장, 공정 유형, 품목 유형
- 프로젝트 상태: `대기`, `진행중`, `완료`, `보류`, `취소`

#### F-002: 10단계 프로세스 관리
| 단계 | 단계명 | 주요 액션 |
|------|--------|-----------|
| 1 | 의뢰 접수 | 접수 폼 작성, 의뢰 내용 기록 |
| 2 | 담당자 지정 | 단계별 담당자 배정 |
| 3 | 고객 협의 | 현장 확인, 반출 기록, 사진 첨부 |
| 4 | 진행 여부 결정 | 진행/보류/취소 결정 |
| 5 | 채권 등록 | 채권 번호 등록, 신규 채권 처리 |
| 6 | 견적 작성 | 템플릿 기반 견적서 생성, 부품 선택 |
| 7 | 제작 | 제작 매뉴얼 작성, 부품 리스트 정리 |
| 8 | 납품/설치 | 설치 매뉴얼 작성, 납품 확인 |
| 9 | 실적 입력 | 실적 데이터 입력 |
| 10 | 최종 문서 정리 | 전체 문서 아카이브, 프로젝트 완료 처리 |

#### F-003: 단계 전환 규칙
- 이전 단계가 완료되어야 다음 단계 진행 가능 (순차 진행)
- 단계 건너뛰기는 관리자만 가능
- 단계 완료 시 자동으로 다음 단계 담당자에게 알림

#### F-004: 유사 프로젝트 복사
- 기존 프로젝트 검색 → 선택 → 복사 생성
- 복사 대상: 기본 정보, 부품 리스트, 견적 항목, 매뉴얼 템플릿
- 복사 후 수정 모드로 진입하여 변경 사항만 편집

### 3.2 문서 관리 (Priority 1)

#### F-005: 단계별 문서 등록
- 각 단계에 해당하는 문서 유형만 등록 가능 (가이드 제공)
- 지원 포맷: PDF, Excel, Word, 한글(HWP), CAD, 이미지(JPG/PNG)
- 파일 크기 제한: 단일 파일 100MB, 프로젝트당 총 2GB

#### F-006: 문서 자동 생성 (템플릿)
- 견적서 템플릿: 프로젝트 정보 + 마스터 부품 선택 → 자동 생성
- 제안서 템플릿: 컨셉, 운전 계획, 사양, 금액 구성 (약 5페이지)
- PDF 다운로드 및 인쇄 지원

#### F-007: 프로젝트별 파일 폴더
- 프로젝트 생성 시 `/프로젝트명/단계별/` 폴더 자동 생성
- 파일 버전 관리 (동일 파일명 업로드 시 버전 자동 증가)

### 3.3 검색 및 조회 (Priority 2)

#### F-008: 통합 검색
- 검색 대상: 프로젝트명, 고객사, 사업장, 공정, 품목, 담당자
- 자동완성 제안 기능
- 검색 결과에서 프로젝트 상세 및 문서 바로 접근

#### F-009: 필터링 조회
- 상태별: 진행중 / 완료 / 보류
- 사업장별, 공정별, 품목별, 담당자별 필터
- 날짜 범위 필터 (등록일, 완료 예정일)

#### F-010: 유사 프로젝트 추천
- 신규 프로젝트 생성 시 사업장+공정+품목 기준 유사 프로젝트 자동 추천
- 유사도 기반 정렬

### 3.4 마스터 데이터 관리 (Priority 3)

#### F-011: 코드 관리
- 관리 항목: 사업장, 공정 유형, 품목 유형, 부품 스펙, 고객사
- CRUD + 사용/미사용 상태 관리
- 견적·제작 시 드롭다운 선택 제공

### 3.5 사용자 및 권한 (Priority 3)

#### F-012: 역할 기반 접근 제어 (RBAC)
| 역할 | 조회 | 생성 | 수정 | 삭제 | 관리 |
|------|------|------|------|------|------|
| ADMIN | ✅ | ✅ | ✅ | ✅ | ✅ |
| MANAGER | ✅ | ✅ | ✅ | ❌ | ❌ |
| USER | 담당만 | ✅ | 담당만 | ❌ | ❌ |
| VIEWER | ✅ | ❌ | ❌ | ❌ | ❌ |

#### F-013: 변경 이력 (Audit Log)
- 모든 데이터 변경(생성/수정/삭제)에 대한 로그 기록
- 기록 항목: 일시, 사용자, 변경 대상, 변경 전/후 값

### 3.6 알림 및 대시보드 (Priority 4)

#### F-014: 알림
- 단계 전환 시 다음 담당자에게 이메일/인앱 알림
- 업무 배정 시 알림
- 알림 이력 조회

#### F-015: 대시보드
- 진행중 프로젝트 현황 (단계별 분포)
- 담당자별 업무량
- 최근 등록/완료 프로젝트
- 월별 실적 요약

#### F-016: My 할일
- 로그인한 사용자에게 배정된 진행중 업무 목록
- 단계별 필요 액션 안내
- 우선순위 표시

## 4. 페이지 구성

| # | 페이지 | 경로 | 설명 |
|---|--------|------|------|
| 1 | 로그인 | `/login` | 이메일/비밀번호 또는 구글 로그인 |
| 2 | 대시보드 | `/dashboard` | 전체 현황 요약, My 할일 |
| 3 | 프로젝트 목록 | `/projects` | 필터·검색·정렬 가능한 목록 |
| 4 | 프로젝트 상세 | `/projects/[id]` | 10단계 진행 현황 + 문서 목록 |
| 5 | 프로젝트 생성 | `/projects/new` | 신규 생성 폼 + 유사 프로젝트 추천 |
| 6 | 견적서 작성 | `/projects/[id]/estimate` | 템플릿 기반 견적서 생성 |
| 7 | 문서 관리 | `/projects/[id]/documents` | 단계별 문서 업로드·조회 |
| 8 | 마스터 관리 | `/admin/master` | 코드 관리 (사업장, 공정 등) |
| 9 | 사용자 관리 | `/admin/users` | 사용자 CRUD, 역할 설정 |
| 10 | 검색 | `/search` | 통합 검색 결과 |
| 11 | 설정 | `/settings` | 프로필, 알림 설정 |
