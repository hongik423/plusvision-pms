# 플러스비젼 PMS - Design Guide

> **버전:** 1.0 | **작성일:** 2026-02-24

---

## 1. 디자인 원칙

### 1.1 핵심 원칙
1. **단순함 우선** — 과도한 기능보다 실무에서 빠르게 사용할 수 있는 직관적 UI
2. **한눈에 파악** — 프로젝트 현황과 내 할일을 즉시 인지 가능
3. **최소 클릭** — 핵심 업무까지 3클릭 이내 도달
4. **일관성** — 전체 페이지에서 동일한 패턴과 컴포넌트 사용

### 1.2 대상 사용자 특성
- 40~50대 제조업 엔지니어 중심
- IT 시스템 사용에 익숙하지 않은 사용자 포함
- 현장에서 태블릿으로도 접근 가능해야 함

→ **큰 글씨, 넉넉한 여백, 명확한 레이블, 아이콘+텍스트 병기**

```

## 5. 주요 컴포넌트 디자인

### 5.1 10단계 스텝퍼 (Step Indicator)
```
핵심 UI 컴포넌트 — 프로젝트 상세 페이지 상단에 항상 표시

수평 배치 (데스크톱):
[●]───[●]───[●]───[◉]───[○]───[○]───[○]───[○]───[○]───[○]
 1     2     3    4(현재) 5     6     7     8     9    10
접수  담당  협의  결정  채권  견적  제작  납품  실적  정리

● 완료 (녹색 체크)     — bg-green-500 + 체크 아이콘
◉ 현재 진행 (파란색)   — bg-blue-500 + 맥동 애니메이션
○ 미진행 (회색)        — bg-gray-200

세로 배치 (모바일/태블릿):
동일 로직, 세로 타임라인으로 전환
각 단계 클릭 시 상세 정보 패널 확장
```

### 5.2 프로젝트 카드
```
┌─────────────────────────────────────────────────┐
│ [진행중]                        2026.02.24       │
│                                                 │
│ CMP 케이블 개조 프로젝트 #2024-001              │
│                                                 │
│ 📍 기흥  |  ⚙️ CMP  |  🔌 케이블               │
│                                                 │
│ 현재 단계: [6단계 — 견적 작성]                  │
│ ■■■■■■□□□□  60%                                │
│                                                 │
│ 담당: 김○○                    → 상세보기        │
└─────────────────────────────────────────────────┘

- 상태 배지: 좌상단, 색상으로 구분
- 프로젝트명: 카드 제목 (18px Bold)
- 분류 태그: 아이콘+텍스트 (사업장, 공정, 품목)
- 진행률 바: 10단계 중 완료 비율
- 담당자: 우하단
```

### 5.3 데이터 테이블
```
- 열 너비 자동 조정, 최소 너비 보장
- 행 높이: 56px (클릭 영역 확보)
- 호버 시 배경색 변경 (gray-50)
- 상태 열: 컬러 배지 (칩 형태)
- 페이지네이션: 20건 단위
- 빈 상태: 일러스트 + "등록된 프로젝트가 없습니다" 문구
```

### 5.4 필터 패널
```
┌─ 필터 ──────────────────────┐
│                              │
│ 상태   [전체▾]              │
│ 사업장 [전체▾]              │
│ 공정   [전체▾]              │
│ 품목   [전체▾]              │
│ 담당자 [전체▾]              │
│ 기간   [시작일] ~ [종료일]  │
│                              │
│ [필터 적용]  [초기화]        │
└──────────────────────────────┘

- 좌측 사이드바 또는 상단 인라인 배치
- 드롭다운 선택, 다중 선택 지원
- 적용된 필터는 칩 형태로 상단에 표시
```

### 5.5 문서 업로드 영역
```
┌ - - - - - - - - - - - - - - - - - - ┐

- 업로드 진행률 표시
- 성공/실패 토스트 알림

## 7. 인터랙션 가이드

### 7.1 버튼
| 유형 | 용도 | 스타일 |
|------|------|--------|
| Primary | 주요 액션 (생성, 저장) | bg-blue-500, text-white, h-44px |
| Secondary | 보조 액션 (취소, 목록) | border-gray-300, text-gray-700, h-44px |
| Danger | 삭제, 취소 | bg-red-500, text-white, h-44px |
| Ghost | 부가 액션 | text-gray-500, hover:bg-gray-100 |

### 7.2 폼 요소
- 입력 필드 높이: 44px (터치 친화적)
- 라벨: 입력 필드 상단, 14px SemiBold
- 필수 항목: 라벨 우측에 빨간 * 표시
- 오류 메시지: 입력 필드 하단, 빨간색 12px
- 드롭다운: 마스터 데이터 연동, 검색 가능한 선택 박스

### 7.3 알림/토스트
- 위치: 화면 우상단
- 자동 사라짐: 5초 후
- 유형별 색상: 성공(녹색), 경고(노란), 오류(빨간), 정보(파란)

### 7.4 모달/다이얼로그
- 삭제 등 위험 액션: 확인 모달 필수
- 모달 너비: sm(400px) / md(600px) / lg(800px)
- 배경: 반투명 오버레이 (black/50%)
