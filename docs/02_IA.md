## 2. 데이터 구조 관계도 (Entity Relationships)

```
[User] 1──N [ProjectMember]
  │                │
  │            N──1 [Project] 1──N [ProjectStage]
  │                │                    │
  │                │               N──1 [StageDocument]
  │                │
  │                ├── 1──N [Estimate]
  │                │         └── 1──N [EstimateItem]
  │                │                    │
  │                │                N──1 [PartSpec] (마스터)
  │                │
  │                ├── 1──N [Manual]
  │                │
  │                └── 1──N [AuditLog]
  │
  ├── [Notification] N──1
  │
  └── [Role] (ADMIN, MANAGER, USER, VIEWER)

[MasterCode]
  ├── Site (사업장)
  ├── ProcessType (공정 유형)
  ├── ItemType (품목 유형)
  ├── Customer (고객사)
  └── PartSpec (부품 스펙)
```

## 4. 화면 흐름도 (User Flow)

### 4.1 신규 프로젝트 생성 흐름
```
대시보드 → [+ 새 프로젝트] 버튼
  → 프로젝트 생성 폼
    → (선택) 유사 프로젝트 검색 → 복사 생성
    → 기본 정보 입력 (고객사, 사업장, 공정, 품목)
    → 담당자 지정 (단계별)
    → [생성] 클릭
  → 프로젝트 상세 페이지 (1단계: 의뢰 접수)
```

### 4.2 단계 진행 흐름
```
프로젝트 상세 → 현재 단계 클릭
  → 단계 상세 패널 열기
    → 필요 문서 업로드
    → 필요 정보 입력
    → [단계 완료] 클릭
  → 다음 단계 자동 활성화
  → 다음 담당자에게 알림 발송
```

### 4.3 견적서 작성 흐름
```
프로젝트 상세 → 견적 탭
  → [견적서 작성] 클릭
    → 프로젝트 정보 자동 입력
    → 부품 선택 (마스터 데이터 드롭다운)
    → 수량/단가 입력
    → 비고 입력
    → [저장] / [PDF 내보내기]
```
