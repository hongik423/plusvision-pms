# 플러스비젼 시스템 구축 요구조건 정의서

> **회의일자:** 2026년 2월 24일  
> **작성목적:** 플러스비젼 대표와의 현업 인터뷰를 바탕으로 도출한 시스템 요구사항 정리

---

## 1. 배경 및 현황

### 1.1 회사 업무 특성
- 주요 고객: 삼성전자 (기흥, 화성, 천안, 평택 사업장)
- 주요 업무: 반도체 공정 설비 관련 제작, 수리, 설치 (케이블 개조, 컨트롤 시스템, 도어락 시스템 등)
- 공정 유형: CMP, CVD, IMP 등 다수

### 1.2 현재 시스템 문제점
- 파일이 Google Drive에 산재하여 빠른 검색·조회 어려움
- 프로젝트 담당자가 퇴사 시 업무 데이터 손실 위험
- 신규 의뢰 접수 시 유사 프로젝트 참조가 어렵고 반복 작업 발생
- 단계별 담당자 지정이 명확하지 않아 책임 소재 불분명
- 데이터 삭제·수정에 대한 권한 관리 미비
- 매뉴얼 부재로 인한 인수인계 시 업무 공백 발생

### 1.3 핵심 목표 (대표 직접 언급)
> "문서를 관리하고, 담당자가 자기 할 일을 알게 하고, 나중에 필요할 때 찾아볼 수 있게 하는 것 - 이 3가지가 큰 목적"

| # | 핵심 목표 | 설명 |
|---|-----------|------|
| 1 | 문서 중앙 관리 | 프로젝트 관련 문서를 체계적으로 축적·보관 |
| 2 | 업무 명확화 | 각 담당자가 현재 자신이 해야 할 일을 인지 |
| 3 | 정보 재활용 | 유사 프로젝트 발생 시 이전 데이터를 빠르게 참조하여 시간 단축 |

---

## 2. 업무 프로세스 정의

### 2.1 프로젝트 표준 10단계 프로세스

```
[1단계] 의뢰 접수
    ↓
[2단계] 담당자 지정
    ↓
[3단계] 고객(삼성 담당자) 협의 → 현장 확인 / 반출 기록
    ↓
[4단계] 진행 여부 결정
    ↓
[5단계] 채권 등록 / 신규 채권 처리
    ↓
[6단계] 견적 작성 (기본 가이드라인 기반)
    ↓
[7단계] 제작 (제작 매뉴얼 작성, 부품 리스트 정리)
    ↓
[8단계] 납품 / 설치 (설치 매뉴얼 작성)
    ↓
[9단계] 실적 입력
    ↓
[10단계] 최종 문서 정리
```

### 2.2 단계별 산출 문서

| 단계 | 생성 문서 |
|------|-----------|
| 의뢰 접수 | 접수 폼, 의뢰 내용 기록 |
| 현장 확인 | 현장 사진, 반출 기록 |
| 견적 | 견적서 |
| 제안 | 제안서 |
| 제작 | 제작 매뉴얼, 부품(파트) 리스트 |
| 설치 | 설치 매뉴얼 |
| 완료 | 실적 데이터, 전체 문서 아카이브 |

---

## 3. 기능 요구사항

### 3.1 프로젝트 관리

#### 3.1.1 프로젝트 등록 및 단계 관리
- 신규 프로젝트 생성 시 표준 10단계가 자동 생성됨
- 각 단계별 담당자를 지정할 수 있음
- 현재 진행 단계와 완료 여부를 한눈에 확인 가능한 진행 현황 화면 제공
- 단계별 날짜(시작일, 완료일) 기록 가능

#### 3.1.2 프로젝트 분류 체계 (표준화)
- **사업장:** 기흥 / 화성 / 천안 / 평택
- **고객:** 삼성전자 (사업부명)
- **공정 유형:** CMP / CVD / IMP / 기타
- **품목 유형:** 케이블 / 컨트롤 시스템 / 도어락 / 디스플레이 / 기타
- 분류 항목은 관리자가 추가·수정 가능 (마스터 코드 관리)

#### 3.1.3 유사 프로젝트 복사 기능
- 신규 프로젝트 생성 시 기존 유사 프로젝트를 검색하여 복사 기능 제공
- 복사된 프로젝트에서 변경 필요한 항목만 수정하여 빠른 견적·제안 작성 가능
- 유사 프로젝트의 부품 리스트, 매뉴얼, 견적서 등 재활용 가능

---

### 3.2 문서 관리

#### 3.2.1 문서 종류
- 제안서, 견적서, 제작 매뉴얼, 설치 매뉴얼, 부품(파트) 리스트, 현장 사진, 도면, 회의록

#### 3.2.2 문서 연동
- 각 문서는 해당 프로젝트의 특정 단계에 연결되어 저장
- 프로젝트 조회 시 해당 단계별 문서 목록 즉시 확인 가능

#### 3.2.3 문서 자동화 (템플릿 기반)
- 제안서·견적서는 기본 템플릿을 기반으로, 프로젝트 정보 입력 시 자동 생성
- 제안서 주요 구성: 컨셉, 운전 계획, 사양, 금액 등 (약 5페이지 기준)
- 자주 쓰는 부품/스펙 항목을 마스터 데이터에서 선택 입력 → 반복 입력 시간 단축

#### 3.2.4 파일 저장 및 첨부
- 사진, 도면(CAD 파일 등), PDF, Excel, 한글 등 다양한 포맷 파일 첨부 가능
- 프로젝트별 파일 폴더 구조 자동 생성

---

### 3.3 검색 및 조회

#### 3.3.1 키워드 검색
- 프로젝트명, 품목 유형, 고객사, 사업장, 공정 유형 등 키워드로 빠른 검색
- 검색 결과에서 해당 프로젝트의 관련 문서·사진 즉시 조회 가능

#### 3.3.2 프로젝트 목록 조회
- 전체 프로젝트 리스트 조회 (진행 중 / 완료 / 전체)
- 단계별 현황 조회 (예: "현재 제작 단계에 있는 프로젝트")
- 담당자별 업무 현황 조회

---

### 3.4 마스터 데이터 관리 (코드 관리)

- 자주 사용되는 부품 스펙(예: 케이블 규격, 디스플레이 사이즈, 전원 사양 등)을 마스터 테이블로 관리
- 견적·제작 단계에서 항목 선택 시 드롭다운 리스트로 제공
- 신규 스펙 발생 시 관리자가 마스터 데이터에 추가 가능
- 표준화되지 않은 기존 데이터의 정규화 작업 필요 (초기 데이터 정비)

---

### 3.5 담당자 관리 및 알림

- 단계별 담당자 지정 기능
- 본인에게 배정된 업무 및 현재 진행 단계 확인 가능 (My 할 일 화면)
- 다음 단계로 업무 이관 시 담당자에게 알림 (이메일 또는 내부 알림)

---

### 3.6 권한 관리 및 데이터 보호

| 권한 등급 | 권한 내용 |
|-----------|-----------|
| 관리자 | 전체 데이터 조회/수정/삭제, 사용자 권한 설정, 마스터 코드 관리 |
| 일반 사용자 | 담당 프로젝트 조회/입력/수정, 삭제 불가 |
| 조회 전용 | 읽기만 가능 (외부 공유용) |

- 데이터 삭제는 관리자만 가능 (일반 사용자 삭제 불가)
- 데이터 변경 이력(로그) 기록
- 정기 백업 기능 (또는 클라우드 자동 백업)

---

### 3.7 인수인계 지원 기능

- 프로젝트별 작업 매뉴얼 및 설치 매뉴얼 등록·열람 가능
- 담당자 교체 시 이전 담당자의 데이터·문서 그대로 인계
- 새 담당자가 기존 진행 상황과 관련 문서를 쉽게 파악 가능한 단계별 화면 제공

| 항목 | 요구사항 |

## 5. 우선순위 정리

| 우선순위 | 기능 | 비고 |
|----------|------|------|
| 1순위 | 프로젝트 10단계 관리 화면 | 핵심 업무 흐름 |
| 1순위 | 단계별 문서(제안서·견적서·매뉴얼) 등록·조회 | 문서 관리 |
| 2순위 | 키워드 검색 및 유사 프로젝트 조회 | 재활용·시간 단축 |
| 2순위 | 유사 프로젝트 복사하여 신규 생성 | 반복 작업 감소 |
| 3순위 | 권한 관리 및 데이터 보호 | 안정성 확보 |
| 3순위 | 마스터 코드 기반 자동 입력·문서 자동화 | 고도화 단계 |
| 4순위 | 담당자 알림, 업무 현황 대시보드 | 편의성 향상 |

## 6. 참고 사항

- 현재 Google Drive + Excel 기반으로 프로세스가 약 80% 구축되어 있으며, 실행·정착이 완전히 이루어지지 않은 상태
- 초기 시스템 구축 시 기존 Google Drive 데이터를 정규화하여 마이그레이션 작업 필요
- 직원들의 시스템 사용 정착이 가장 중요한 성공 요소임을 대표가 강조
- 과도한 관리 기능보다 실무에서 빠르게 사용할 수 있는 **단순하고 실용적인 시스템** 지향
